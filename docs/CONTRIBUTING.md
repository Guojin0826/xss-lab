# XSS漏洞演示靶场 - 贡献指南

感谢您对本项目的关注！我们欢迎所有形式的贡献。

## 📋 行为准则

- 本项目仅用于教育目的
- 请勿提交任何恶意代码或攻击性内容
- 尊重所有贡献者
- 保持专业和友好的交流

## 🐛 报告Bug

如果您发现了Bug，请通过[GitHub Issues](https://github.com/Guojin0826/xss-lab/issues)提交：

1. 使用清晰的标题描述问题
2. 详细描述复现步骤
3. 提供预期的行为和实际行为
4. 如果可能，提供截图或错误日志
5. 说明您的环境（PHP版本、Web服务器等）

## 💡 提出新功能

如果您有新功能的想法：

1. 先在Issues中搜索，确保没有重复的建议
2. 创建新的Issue，详细描述功能
3. 说明为什么这个功能有用
4. 如果可能，提供实现思路

## 🔧 提交代码

### 开发环境设置

1. Fork本项目
2. Clone到本地
```bash
git clone https://github.com/Guojin0826/xss-lab.git
cd xss-lab
```

3. 创建特性分支
```bash
git checkout -b feature/your-feature-name
```

### 代码规范

**PHP代码规范：**
- 遵循PSR-12编码标准
- 使用有意义的变量名和函数名
- 添加必要的注释
- 保持代码简洁清晰

**HTML/CSS规范：**
- 使用语义化HTML标签
- CSS类名使用小写和连字符
- 保持良好的缩进和格式

**JavaScript规范：**
- 使用ES6+语法
- 变量使用const/let，避免var
- 函数和变量使用驼峰命名

### 提交规范

使用清晰的提交信息：

```
类型: 简短描述

详细描述（可选）
```

类型包括：
- `feat`: 新功能
- `fix`: Bug修复
- `docs`: 文档更新
- `style`: 代码格式调整
- `refactor`: 代码重构
- `test`: 测试相关
- `chore`: 构建/工具相关

示例：
```
feat: 添加XSS过滤演示功能

- 添加输入过滤示例
- 添加输出编码示例
- 更新README文档
```

### Pull Request流程

1. 确保代码通过测试（如果有）
2. 更新相关文档
3. 提交Pull Request
4. 在PR描述中：
   - 说明更改的内容
   - 关联相关的Issue
   - 提供测试步骤

### 代码审查

所有PR都需要经过审查：
- 确保代码质量
- 检查安全性
- 验证功能正确性
- 检查文档完整性

## 📚 文档贡献

文档改进同样重要：
- 修正拼写或语法错误
- 改进文档结构
- 添加缺失的文档
- 更新过时的信息
